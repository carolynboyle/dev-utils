# dev-utils

A collection of standalone developer tools for workflow automation. Each tool
is designed to work independently and can be composed or imported by other
projects. The collection currently includes Python packages, bash utilities,
and container configurations.

---

## Tools

### Python

**setupkit** — Plugin lifecycle manager.
- Installs, updates, and checks the status of Python packages without manual
  pip paths or version tracking. Reads package metadata from `manifest.fletch`
  files generated by [fletcher](https://github.com/carolynboyle/doc-gen) and
  [doc-gen](https://github.com/carolynboyle/doc-gen). Depends on the
  doc-gen/fletcher pipeline; other dev-utils packages do not.

**contactkit** — Multi-format contact import and CLI interface.
- Parses CSV/VCF files from Gmail, Proton, Outlook, and Apple; inserts into a
  contacts schema via dbkit; logs imports to a `contact_imports` audit table.
  Designed to work with multiple database backends. Gmail plugin complete;
  others in progress.

**dbkit** — Database abstraction layer.
- Provides a consistent connection and query interface for PostgreSQL (via
  psycopg3) and SQLite. Credentials via `~/.pgpass` for PostgreSQL; rows
  returned as dicts.

**fletcher** — Raw GitHub URL generator.
- Reads `manifest.yml` files produced by
  [doc-gen](https://github.com/carolynboyle/doc-gen) and outputs
  `manifest.fletch` files containing raw GitHub URLs for every file in the
  project tree.

**mcpkit** — MCP tool executor. *(Not yet ready for use.)*
- Registers and dispatches tool calls via the Model Context Protocol. Plugin
  architecture for adding new tool handlers.

**menukit** — YAML-driven CLI menu library.
- Provides `MenuBuilder` and `InputPlugin` base classes for building consistent
  interactive menus across CLI tools.

**todo** — JSON-backed todo list manager.
- Standalone CLI tool; stores tasks in `~/.local/share/todo/todos.json` and
  exports a `TODO.md` to the current working directory on exit.

**viewkit** — YAML-driven SQL view and query builder.
- Defines and executes database views and queries from YAML configuration.
  Includes an on-the-fly query runner for ad hoc use.

### Bash

Utility scripts in `bash/` organized by purpose:

- `connection_tools/` — session management helpers (e.g. `tmux-launch.sh`)
- `container_tools/` — container build and management scripts
- `templates/` — starter templates for new scripts

### Containers

Docker Compose configurations in `containers/` for development services:

- `adminer/` — lightweight database browser
- `pgadmin/` — PostgreSQL administration UI

## Setup

All Python packages can be installed individually with a standard pip install:

```bash
cd python/<package>
pip install .
```

To bootstrap the full tools venv and install setupkit (which manages
subsequent installs via the doc-gen/fletcher pipeline):

```bash
bash setup.sh
```

Once setupkit is installed:

```bash
setupkit init <package>
setupkit install <package>
```
