# setupkit

Plugin installer for dev-utils / Project Crew.

Installs and updates Project Crew plugins by reading plugin configs from
`~/.config/dev-utils/setupkit/`, fetching upstream
[fletcher](https://github.com/carolynboyle/dev-utils/tree/master/python/fletcher)
manifests to check versions, and running pip install as needed.

Intended for use by Quartermaster (orchestration) or directly from the command line.

---

## Requirements

- Python 3.11 or later
- `pyyaml >= 6.0`
- `requests >= 2.31.0`
- `packaging >= 23.0`

---

## Installation

```bash
pip install -e ~/projects/dev-utils/python/setupkit
```

---

## Quick start

```bash
# Generate a plugin config interactively
setupkit init dbkit

# Install it
setupkit install dbkit

# Check all plugins for updates
setupkit check

# Install all plugins
setupkit install
```

---

## How it works

Each plugin has a config yaml in `~/.config/dev-utils/setupkit/`:

```
~/.config/dev-utils/setupkit/
├── dbkit.yaml
├── viewkit.yaml
├── menukit.yaml
└── fletcher.yaml
```

These are generated by `setupkit init` and declare where to find, verify,
and install each plugin. setupkit fetches the plugin's `manifest.fletch`
from GitHub to check the upstream version, then calls pip install if an
update is needed.

---

## CLI reference

### `setupkit init <name>`

Interactively generate a plugin config yaml. Fetches the upstream
`manifest.fletch`, presents the file list for path selection, infers the
`pyproject.toml` path, and writes the result to
`~/.config/dev-utils/setupkit/<name>.yaml`.

If a config already exists, shows the existing content and prompts before
overwriting.

```bash
setupkit init dbkit
```

### `setupkit install [name] [--force]`

Install or update a plugin. Fetches `manifest.fletch`, compares the upstream
version against what is installed, and runs pip install if needed.

Omit `name` to install all configured plugins.

```bash
setupkit install dbkit          # install or update dbkit
setupkit install dbkit --force  # reinstall even if up to date
setupkit install                # install or update all plugins
setupkit install --force        # reinstall all plugins
```

### `setupkit check [name]`

Check whether a plugin needs installing or updating without making any changes.

Omit `name` to check all configured plugins.

```bash
setupkit check dbkit   # check dbkit only
setupkit check         # check all plugins
```

## Plugin config format

Plugin configs are generated by `setupkit init` and live in
`~/.config/dev-utils/setupkit/`. They can also be edited manually.

```yaml
# setupkit plugin config — generated by setupkit init
# Edit manually or regenerate with: setupkit init dbkit

name: dbkit
version: 0.1.0
manifest_url: https://raw.githubusercontent.com/carolynboyle/dev-utils/master/.doc-gen/manifest.fletch
pyproject: python/dbkit/pyproject.toml
path_prefix: python/dbkit/dbkit/
install:
  method: pip
  url: git+https://github.com/carolynboyle/dev-utils.git#subdirectory=python/dbkit
```

| Field | Description |
|---|---|
| `name` | Package name as declared in `pyproject.toml` |
| `version` | Currently installed version — updated by setupkit after install |
| `manifest_url` | Raw URL to the repo's `manifest.fletch` file |
| `pyproject` | Path within the manifest to the plugin's `pyproject.toml` |
| `path_prefix` | Path prefix to filter manifest entries to this plugin's source files |
| `install.method` | Install method — currently only `pip` is supported |
| `install.url` | URL passed to pip (git+ URL for GitHub packages) |

## Logging

setupkit writes to `~/.local/share/dev-utils/setupkit.log` at INFO level.
Warnings and errors also print to stderr.

```
2026-04-16 14:32:01 [INFO] Installing plugin: dbkit
2026-04-16 14:32:01 [INFO] updated dbkit (installed: 0.1.0, upstream: 0.2.0)
2026-04-16 14:32:04 [INFO] dbkit 0.2.0 updated successfully
```

## Public API

setupkit can be used as a library by Quartermaster or other tools:

```python
from setupkit import install_plugin, check_plugin, check_all, install_all

# Check a single plugin
result = check_plugin("dbkit")
print(result.message)
# dbkit 0.1.0 is up to date

# Install all plugins
results = install_all()
for r in results:
    print(r.message)

# Check for updates across all plugins
results = check_all()
for r in results:
    if r.version_info and r.version_info.update_available:
        print(f"Update available: {r.message}")
```

### `init_plugin(name)`

Run the interactive init flow for a named plugin. Writes config to
`~/.config/dev-utils/setupkit/<name>.yaml`.

### `install_plugin(name, force=False) -> InstallResult`

Install or update a single plugin by name. Returns an `InstallResult`.

### `install_all(force=False) -> list[InstallResult]`

Install or update all configured plugins. Continues past individual
failures, collecting results.

### `check_plugin(name) -> InstallResult`

Check whether a plugin needs installing or updating. Does not modify anything.

### `check_all() -> list[InstallResult]`

Check all configured plugins. Continues past individual failures.
