### sniffkit — New dev-utils Package

New package built from scratch in `dev-utils/python/sniffkit/`.
Content-type detector and file classifier for raw `.txt` output files.
Primary consumer: designing-gemma experiment results.

**Status: all five detectors registered and working.**

Structure scaffolded with treekit (worked perfectly first try).
`pip install -e .` succeeded. Registry verified:

```bash
python -c "from sniffkit.detectors import get_registry; print(list(get_registry().keys()))"
['css', 'html', 'json', 'md', 'sql']
```

Files completed:
- `pyproject.toml` ✅
- `src/sniffkit/detectors/__init__.py` — protocol, registry, @register decorator ✅
- `src/sniffkit/detectors/css_detector.py` ✅
- `src/sniffkit/detectors/html_detector.py` ✅

**Next step:** smoke test the CLI:


instance. `classifier.py` calls `detector_cls().detect(text)` — instantiates
per call. If this becomes a performance issue with large result sets,
cache instances in `ResultClassifier.__init__`.

### Card View for Experiment Browser — Designed

Decision: card view replaces the NONE/DISABLED stub buttons in the runs
detail panel. Same toggle pattern as the experiment list header (list/cards).

- HTML output → scaled iframe preview
- CSS output → styled code block or swatch
- No classified output → metadata card only
- Click card → Flask-rendered run detail page at `/runs/<experiment_id>/<run_id_string>`
- Run detail page: mesh = CRUD (notes field editable), non-mesh = read-only
